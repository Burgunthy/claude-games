# Claude Poker - Project Overview for LLMs

## Summary
Claude Poker is a comprehensive Texas Hold'em poker game built for the terminal using Ink (React for CLI). It features AI vs AI spectator mode, visible AI thoughts, multiple themes, and a complete ranking system.

## Key Technologies
- **UI Framework**: Ink (React for CLI) with @inkjs/ui components
- **Language**: TypeScript 5.3+ with strict mode
- **Storage**: JSON files in `~/.claude-games/claude-poker/`
- **Testing**: Vitest
- **Node**: 18.0.0+

## Architecture Overview

### Core Engine (`src/core/`)
- **types.ts**: All TypeScript interfaces and enums (Card, Player, GameState, etc.)
- **engine.ts**: PokerEngine class extending EventEmitter, handles game flow
- **hand-evaluator.ts**: Custom hand evaluation (Royal Flush to High Card)
- **pot-calculator.ts**: Side pot calculation for all-in situations
- **deck.ts**: Deck creation, shuffling, dealing
- **constants.ts**: Game constants (blinds, XP rewards, tier thresholds)

### AI System (`src/ai/`)
- **base.ts**: BaseAI abstract class with decision-making interface
- **strategies/**: Four AI implementations (easy, normal, hard, maniac)
- **thoughts.ts**: Thought formatting and display system

AI decisions have 3 stages:
1. Perceive (📊): See cards, position, pot
2. Analyze (🧮): Calculate strength, pot odds, draws
3. Decide (💭): Choose action with reasoning

### UI Components (`src/ui/`)
- **components/**: Reusable React components (Table, ActionButtons, ThoughtBubble, GameLog)
- **screens/**: Main screens (MainMenu, GameScreen, AiOnlyScreen, StatsScreen)
- **themes/**: Three visual themes (casino, modern, retro)
- **art/**: ASCII art for cards and titles

### Storage (`src/storage/`)
- **player-data.ts**: Load/save player progress, XP, tier
- **settings.ts**: Game settings (theme, language, difficulty)
- **history.ts**: Game history and statistics

### i18n (`src/i18n/`)
- JSON translation files for English, Korean, Japanese
- Simple translation helper with `t(key, ...args)` function

### Ranking (`src/ranking/`)
- **tiers.ts**: Tier definitions and progress calculation
- **calculator.ts**: XP rewards and tier-up logic

## Game Flow

1. Menu screen → User selects mode
2. Game setup → Create PokerEngine with config
3. Hand loop:
   - Deal hole cards (2 each)
   - Post blinds
   - Betting rounds (preflop → flop → turn → river)
   - Showdown (if >1 player remains)
   - Award pot
4. Repeat until tournament ends or user quits

## AI Decision Making

Each AI strategy implements:
```typescript
decideAction(holeCards, communityCards, context): Decision
generateThoughts(holeCards, communityCards, context, decision): Thought[]
```

Context includes: phase, pot, currentBet, potOdds, position, playersInHand

## Hand Evaluation

Custom implementation from scratch (no external library):
- Groups cards by rank and suit
- Checks for flush, straight, pairs, etc.
- Returns numeric score for easy comparison
- Handles kickers for tiebreaking

## Side Pots

When players go all-in with different amounts:
1. Create side pots for each all-in level
2. Players can only win pots they contributed to
3. Distribute correctly at showdown

## Development Guidelines

1. **Type Safety**: All types defined in `src/core/types.ts`
2. **Event-Driven**: Engine emits events for UI to react
3. **No UI in Core**: Game engine is UI-independent
4. **Storage**: Use existing storage helpers, don't touch filesystem directly
5. **i18n**: Always use `t()` function for user-facing strings
6. **Testing**: Scenario tests in `tests/scenarios/`

## Common Patterns

### Adding a new AI strategy
1. Extend BaseAI in `src/ai/strategies/`
2. Implement `decideAction()` and `generateThoughts()`
3. Export from `src/ai/index.ts`
4. Add to AIStrategyType in types.ts

### Adding a new theme
1. Create theme file in `src/ui/themes/`
2. Export Theme object with colors, cardArt, borders
3. Add to THEMES map in `src/ui/themes/index.ts`

### Adding a new translation
1. Create JSON file in `src/i18n/`
2. Add to language list in `src/i18n/index.ts`
3. Follow existing key structure

## Known Limitations

- Hand evaluation is custom (not using poker-ts library)
- No multiplayer/network play
- AI doesn't learn from past hands
- No tournament modes (single table only)
